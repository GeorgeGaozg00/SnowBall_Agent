1. 项目概述
================================================================================

1.1 项目背景
------------
雪球智能评论助手是一个基于AI大模型的自动化评论工具，旨在帮助用户自动对雪球
平台上的热门帖子、推荐帖子和关注用户的帖子进行智能评论或点赞。该工具通过
集成多个主流AI大模型（火山引擎、OpenAI、百度文心一言、阿里通义千问），
自动生成高质量的评论内容，并自动发布到雪球平台。

1.2 项目目标
------------
- 提高用户在雪球平台的互动效率
- 自动生成高质量、个性化的评论内容
- 支持多种AI大模型，灵活切换
- 提供友好的Web界面，方便配置和监控
- 支持测试模式，避免误操作

1.3 适用人群
------------
- 雪球平台活跃用户
- 需要批量管理社交互动的用户
- 对AI自动评论感兴趣的开发者

2. 功能需求
================================================================================

2.1 核心功能
------------

2.1.1 热门帖评论
  - 自动抓取雪球热门帖子
  - 使用AI生成评论内容
  - 自动发布评论到帖子
  - 支持设置每日评论限制
  - 支持自定义评论间隔时间

2.1.2 推荐帖评论
  - 自动抓取雪球推荐帖子
  - 使用AI生成评论内容
  - 自动发布评论到帖子
  - 支持设置每日评论限制
  - 支持自定义评论间隔时间

2.1.3 关注者评论/点赞
  - 获取用户关注列表
  - 支持选择性评论特定关注用户
  - 支持两种操作模式：
    * 评论模式：生成AI评论并发布
    * 点赞模式：仅对帖子点赞
  - 支持设置每个用户的帖子处理数量

2.2 配置管理
------------

2.2.1 大模型配置
  - 支持多个AI大模型配置：
    * 火山引擎（Doubao）
    * OpenAI（GPT）
    * 百度文心一言
    * 阿里通义千问
  - 每个模型独立配置API Key
  - 支持通过单选框切换当前使用的模型
  - 配置自动保存到本地文件

2.2.2 用户Cookie配置
  - 配置雪球平台的Cookie信息
  - 支持Cookie有效性验证
  - 配置自动保存

2.2.3 评论设置
  - 每日评论限制设置（1-100条）
  - 评论间隔时间设置（最小10秒，最大600秒）
  - 测试模式开关（测试模式下不实际发布）

2.3 监控功能
------------

2.3.1 实时日志显示
  - 显示任务执行过程
  - 显示文章抓取状态
  - 显示评论生成过程
  - 显示评论发布结果
  - 支持日志清空功能

2.3.2 统计数据
  - 已处理文章数/用户数
  - 成功发布评论数
  - 失败尝试次数
  - 任务运行状态

2.4 任务管理
------------

2.4.1 任务控制
  - 启动/停止热门帖评论任务
  - 启动/停止推荐帖评论任务
  - 启动/停止关注者评论任务
  - 任务互斥机制（同一时间只能运行一个任务）

2.4.2 任务监控
  - 定期更新任务状态
  - 自动检测任务异常
  - 任务完成后自动重置状态

3. 技术架构
================================================================================

3.1 系统架构
------------
采用前后端分离架构：

前端：
  - 技术栈：HTML5 + CSS3 + JavaScript
  - UI框架：Tailwind CSS
  - 图标库：Font Awesome 4.7.0
  - 特点：响应式设计，支持移动端访问

后端：
  - 语言：Python 3.x
  - 框架：Flask
  - 特点：RESTful API设计，支持多线程任务管理

3.2 目录结构
------------
xueqiu-hot-stocks/
├── backend/                    # 后端代码目录
│   ├── app.py                 # Flask主应用
│   ├── commenter.py           # 热门帖/推荐帖评论逻辑
│   ├── following_commenter.py # 关注者评论逻辑
│   ├── following_fetcher.py   # 关注列表获取逻辑
│   └── config.json            # 配置文件
├── test/                       # 测试代码目录
│   └── [各种测试文件]
├── index.html                  # 前端主页面
├── convert_config.py           # 配置文件转换工具
└── .gitignore                  # Git忽略文件配置

3.3 数据流程
------------
1. 用户在前端界面配置API Key和Cookie
2. 前端通过HTTP请求将配置发送到后端
3. 后端保存配置到config.json文件
4. 用户启动任务，前端发送启动请求
5. 后端创建任务线程，开始执行：
   a. 抓取雪球帖子数据
   b. 调用AI大模型生成评论
   c. 发布评论到雪球平台
   d. 通过日志回调更新状态
6. 前端定期轮询状态接口，获取最新日志和统计
7. 任务完成或用户停止，后端清理资源

3.4 多线程设计
--------------
- 主线程：处理HTTP请求，提供API服务
- 任务线程：执行评论任务，避免阻塞主线程
- 线程锁：保护共享资源（日志、统计数据）
- 状态管理：使用全局变量管理任务状态

4. 系统配置
================================================================================

4.1 配置文件格式
----------------
配置文件位于：backend/config.json

{
  "selectedModel": "ark",              // 当前选中的模型
  "models": {                          // 模型配置
    "ark": {                           // 火山引擎
      "apiKey": "your-api-key"
    },
    "openai": {                        // OpenAI
      "apiKey": "your-api-key",
      "baseUrl": "https://api.openai.com/v1"
    },
    "baidu": {                         // 百度文心一言
      "apiKey": "your-api-key",
      "secretKey": "your-secret-key"
    },
    "alibaba": {                       // 阿里通义千问
      "apiKey": "your-api-key",
      "baseUrl": ""
    }
  },
  "xueQiuCookie": "your-cookie",       // 雪球Cookie
  "dailyLimit": 60,                    // 每日评论限制
  "delayMin": 30,                      // 最小间隔时间（秒）
  "delayMax": 120,                     // 最大间隔时间（秒）
  "testMode": false                    // 测试模式
}

4.2 获取雪球Cookie
------------------
1. 登录雪球网站（https://xueqiu.com）
2. 打开浏览器开发者工具（F12）
3. 切换到"Network"标签
4. 刷新页面，找到任意请求
5. 在请求头中找到"Cookie"字段
6. 复制完整的Cookie字符串

4.3 获取AI模型API Key
---------------------
火山引擎：
  - 访问：https://www.volcengine.com/
  - 注册账号并开通Doubao服务
  - 在控制台获取API Key

OpenAI：
  - 访问：https://platform.openai.com/
  - 注册账号并创建API Key
  - 注意：需要科学上网

百度文心一言：
  - 访问：https://cloud.baidu.com/
  - 开通文心一言服务
  - 获取API Key和Secret Key

阿里通义千问：
  - 访问：https://dashscope.aliyun.com/
  - 开通通义千问服务
  - 获取API Key

5. API接口文档
================================================================================

5.1 配置管理接口
----------------

5.1.1 获取配置
  接口：GET /api/config
  说明：获取当前保存的配置信息
  
  响应示例：
  {
    "success": true,
    "exists": true,
    "data": {
      "selectedModel": "ark",
      "models": {...},
      "xueQiuCookie": "...",
      ...
    }
  }

5.1.2 保存配置
  接口：POST /api/config
  说明：保存配置到文件
  
  请求体：
  {
    "selectedModel": "ark",
    "models": {...},
    "xueQiuCookie": "...",
    "dailyLimit": 60,
    "delayMin": 30,
    "delayMax": 120,
    "testMode": false
  }
  
  响应示例：
  {
    "success": true,
    "message": "配置保存成功"
  }

5.2 任务控制接口
----------------

5.2.1 启动热门帖/推荐帖评论
  接口：POST /api/start-commenting
  说明：启动热门帖或推荐帖评论任务
  
  请求体：
  {
    "models": {...},
    "selectedModel": "ark",
    "xueQiuCookie": "...",
    "dailyLimit": 60,
    "delayMin": 30,
    "delayMax": 120,
    "testMode": false,
    "taskType": "hot"  // 或 "recommend"
  }
  
  响应示例：
  {
    "success": true,
    "message": "评论任务已启动"
  }

5.2.2 启动关注者评论
  接口：POST /api/start-following-comments
  说明：启动关注者评论或点赞任务
  
  请求体：
  {
    "selectedUsers": [
      {
        "uid": "123456",
        "screen_name": "用户名",
        ...
      }
    ],
    "postsPerUser": 10,
    "testMode": false,
    "actionType": "comment"  // 或 "like"
  }
  
  响应示例：
  {
    "success": true,
    "message": "关注者评论任务已启动"
  }

5.2.3 停止任务
  接口：POST /api/stop-commenting
  说明：停止当前运行的任务
  
  响应示例：
  {
    "success": true,
    "message": "任务已停止"
  }

5.3 状态查询接口
----------------

5.3.1 获取运行状态
  接口：GET /api/run-status
  说明：获取当前任务的运行状态、统计数据和日志
  
  响应示例：
  {
    "success": true,
    "isRunning": true,
    "taskType": "following",  // 或 "commenting"
    "data": {
      "processedUsers": 5,
      "successComments": 20,
      "failedAttempts": 3,
      "actionType": "comment"
    },
    "logs": [
      {
        "timestamp": "07:30:15",
        "type": "info",
        "message": "正在处理用户..."
      }
    ]
  }

5.4 关注列表接口
----------------

5.4.1 获取关注列表
  接口：POST /api/get-following-list
  说明：获取用户的关注列表
  
  请求体：
  {
    "xueQiuCookie": "..."
  }
  
  响应示例：
  {
    "success": true,
    "data": {
      "totalCount": 100,
      "followingList": [
        {
          "uid": "123456",
          "screen_name": "用户名",
          "description": "用户简介",
          "profile_image": "头像URL"
        }
      ]
    }
  }

5.4.2 检查关注列表缓存
  接口：GET /api/check-following-cache
  说明：检查是否有本地缓存的关注列表
  
  响应示例：
  {
    "success": true,
    "exists": true,
    "data": {
      "totalCount": 100,
      "updateTime": "2026-04-03 07:00:00",
      "followingList": [...]
    }
  }

6. 部署指南
================================================================================

6.1 环境要求
------------
- Python 3.7 或更高版本
- 现代浏览器（Chrome、Firefox、Safari等）
- 网络连接（需要访问雪球API和AI模型API）

6.2 安装步骤
------------

步骤1：克隆代码仓库
  git clone https://github.com/GeorgeGaozg00/SnowBall_Agent.git
  cd SnowBall_Agent

步骤2：安装Python依赖
  pip install flask requests

步骤3：配置系统
  1. 打开浏览器，访问 http://localhost:5001
  2. 点击"大模型配置"，输入AI模型的API Key
  3. 点击"用户Cookie配置"，输入雪球Cookie
  4. 点击"评论设置"，调整评论参数

步骤4：启动服务
  cd backend
  python app.py

步骤5：访问界面
  打开浏览器，访问 http://localhost:5001

6.3 生产环境部署
----------------

使用Gunicorn部署：
  pip install gunicorn
  gunicorn -w 4 -b 0.0.0.0:5001 backend.app:app

使用Nginx反向代理：
  server {
      listen 80;
      server_name your-domain.com;
      
      location / {
          proxy_pass http://127.0.0.1:5001;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
      }
  }

6.4 Docker部署（可选）
----------------------
创建Dockerfile：
  FROM python:3.9-slim
  WORKDIR /app
  COPY requirements.txt .
  RUN pip install -r requirements.txt
  COPY . .
  EXPOSE 5001
  CMD ["python", "backend/app.py"]

构建并运行：
  docker build -t xueqiu-commenter .
  docker run -p 5001:5001 xueqiu-commenter

7. 使用说明
================================================================================

7.1 快速开始
------------

1. 启动服务
   cd backend
   python app.py

2. 打开浏览器
   访问 http://localhost:5001

3. 配置系统
   - 点击左侧菜单"大模型配置"
   - 选择要使用的AI模型
   - 输入对应的API Key
   - 点击"保存配置"

   - 点击左侧菜单"用户Cookie配置"
   - 输入雪球Cookie
   - 点击"保存配置"

4. 启动任务
   - 点击"启动热门帖回复"按钮
   - 或点击"启动推荐帖回复"按钮
   - 或点击"启动关注者评论"按钮

5. 监控任务
   - 查看实时日志
   - 查看统计数据
   - 需要时点击"停止"按钮

7.2 功能详解
------------

7.2.1 热门帖评论
  1. 点击绿色的"启动热门帖回复"按钮
  2. 系统自动抓取热门帖子
  3. 使用AI生成评论
  4. 自动发布评论
  5. 按照设置的间隔时间继续处理下一篇
  6. 达到每日限制或用户停止时结束

7.2.2 推荐帖评论
  1. 点击蓝色的"启动推荐帖回复"按钮
  2. 系统自动抓取推荐帖子
  3. 使用AI生成评论
  4. 自动发布评论
  5. 按照设置的间隔时间继续处理下一篇
  6. 达到每日限制或用户停止时结束

7.2.3 关注者评论/点赞
  1. 点击紫色的"启动关注者评论"按钮
  2. 系统弹出关注者列表
  3. 选择要处理的关注者（可多选）
  4. 设置每个用户的帖子数量
  5. 选择操作类型：
     - 评论：生成AI评论并发布
     - 点赞：仅对帖子点赞
  6. 点击"启动评论"按钮
  7. 系统自动处理每个选中用户的帖子

7.3 测试模式
------------
  测试模式用于验证系统功能，不会实际发布评论或点赞。
  
  开启方法：
  1. 点击"评论设置"
  2. 勾选"测试模式（不实际发布评论）"
  3. 保存配置
  
  效果：
  - 系统会正常抓取帖子
  - 正常生成评论
  - 但不会实际发布到雪球
  - 日志会显示"测试模式：跳过发布评论"

8. 注意事项
================================================================================

8.1 安全注意事项
----------------
1. 保护好您的API Key和Cookie信息
2. 不要将config.json文件上传到公开仓库
3. 定期更换Cookie，避免账号风险
4. 建议使用测试模式验证功能后再实际运行

8.2 使用限制
------------
1. 遵守雪球平台的使用规则
2. 不要设置过高的评论频率，避免被判定为垃圾信息
3. 建议每日评论数量不超过100条
4. 评论间隔时间建议设置在30-120秒之间

8.3 常见问题
------------

Q1: Cookie失效怎么办？
A1: 重新登录雪球网站，获取新的Cookie并更新配置

Q2: AI生成评论质量不高怎么办？
A2: 可以尝试切换不同的AI模型，不同模型的生成效果可能不同

Q3: 任务启动后立即停止怎么办？
A3: 检查API Key是否正确，检查Cookie是否有效

Q4: 如何避免被雪球封号？
A4: 
  - 不要设置过高的评论频率
  - 评论内容要有质量，避免重复
  - 不要在短时间内大量评论
  - 适当使用点赞功能，不要只评论

Q5: 支持哪些AI模型？
A5: 目前支持：
  - 火山引擎（Doubao）
  - OpenAI（GPT系列）
  - 百度文心一言
  - 阿里通义千问

用户点击“撰写投资笔记”按钮，：用户通过雪球首页“投资笔记”入口或个人中心“我的笔记”页面，点击“撰写笔记”按钮，触发撰写功能；→ 弹出历史投资笔记列表窗口：系统自动弹出独立窗口，居中显示用户所有历史投资笔记，按创建时间倒序排列，列表中清晰展示每篇笔记的文章ID、完整标题及内容前150字（超出部分以“...”结尾）；每篇笔记右侧配备专属图标按钮，点击该按钮可弹出新的详情窗口，窗口内完整展示该笔记的文章ID、标题、全部内容，同时在线实时获取并显示该笔记的阅读量、收藏量、点赞数三项数据；→ 用户触发智能生成：用户确认历史笔记信息后，点击列表底部居中的“生成今天的投资笔记”按钮，触发当日笔记生成流程；→ 大模型提交与生成：系统自动筛选近5天（含当日）的所有历史投资笔记（含已发布、未发布、草稿状态），完整打包每篇笔记的ID、标题、内容，同时提交系统默认提示词至指定大模型接口，由大模型结合历史笔记风格及提示词要求，生成当日投资笔记；→ 当日笔记展示：大模型生成完成后，自动弹出查看窗口，展示生成的当日投资笔记，供用户逐字查看、手动编辑修改；→ 用户确认：用户查看并确认当日笔记内容无误（可随时修改）后，点击窗口内“确认发表”按钮，进入发表窗口；→ 发表设置：发表窗口内展示当日笔记最终预览，用户可勾选“是否发表至个人专栏”选项（默认不勾选），完成发表相关设置；→ 完成发布：用户确认设置后，点击窗口底部“发表”按钮，系统提交笔记内容至服务器，完成发布操作。
3.2 流程节点说明

- 触发方式：用户可通过雪球程序首页“投资笔记”入口、个人中心“我的笔记”页面，点击“撰写笔记”按钮触发功能。

- 历史笔记展示：触发撰写后，自动弹出独立窗口，同步加载用户所有历史投资笔记，按时间倒序排列（最新笔记在前）。

- 智能生成：用户点击生成按钮后，系统自动筛选近5天（含当日，若当日已有未发布笔记则不重复筛选）的历史笔记，打包其核心内容（ID、标题、完整内容），提交至指定大模型接口，生成符合用户投资风格、贴合近期投资场景的当日投资笔记。

- 确认与发布：生成的当日笔记展示在独立查看窗口，用户可直接编辑修改；确认无误后进入发表窗口，完成发布操作及专栏同步选择。
