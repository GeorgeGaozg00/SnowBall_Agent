## 项目简介

雪球智能评论助手是一个基于AI大模型的自动化评论工具，帮助您在雪球平台上自动评论热门帖子、推荐帖子和关注用户的帖子，并支持智能生成投资笔记。

### 主要功能

- **热门帖评论**：自动抓取雪球热门帖子，使用AI生成评论并自动发布
- **推荐帖评论**：自动抓取雪球推荐帖子，使用AI生成评论并自动发布
- **关注者评论/点赞**：自动获取关注用户的帖子，选择性进行评论或点赞
- **投资笔记生成**：基于历史投资笔记智能生成今日投资笔记，支持结合专栏文章
- **多AI模型支持**：支持火山引擎、OpenAI、百度文心一言、阿里通义千问、Google Gemini等多个AI大模型
- **测试模式**：在测试模式下验证功能，不会实际发布内容

## 基础配置

### 1. 大模型配置

点击左侧菜单 **"大模型配置"**，弹出配置窗口：

#### 支持的AI模型

| 模型名称 | 说明 | 配置项 |
|---------|------|--------|
| 火山引擎（豆包） | 字节跳动AI模型 | API Key |
| OpenAI | GPT系列模型 | API Key、Base URL |
| 百度文心一言 | 百度AI模型 | API Key、Secret Key |
| 阿里通义千问 | 阿里云AI模型 | API Key、Base URL |
| Google Gemini | Google AI模型 | API Key、Base URL |
| DeepSeek | DeepSeek模型 | API Key、Base URL |
| Claude | Anthropic模型 | API Key、Base URL |

#### 配置步骤

1. **选择模型**：点击对应模型的单选框，选中的模型会高亮显示
2. **填写配置**：
   - 火山引擎：输入API Key
   - OpenAI兼容模型（OpenAI、DeepSeek、Claude）：输入API Key和Base URL
   - 百度文心一言：输入API Key和Secret Key
   - 阿里通义千问：输入API Key（可选Base URL）
   - Google Gemini：输入API Key（可选Base URL）
3. **测试模型**：点击模型卡片上的 **"测试"** 按钮，验证配置是否正确
   - 测试过程中会显示实时操作日志
   - 测试成功显示绿色"成功"标签
   - 测试失败显示红色"失败"标签，鼠标悬停可查看具体错误原因
4. **保存配置**：点击 **"保存配置"** 按钮

#### 获取API Key的方法

- **火山引擎**：访问 https://www.volcengine.com/ 注册并开通服务
- **OpenAI**：访问 https://platform.openai.com/ 创建API Key
- **百度文心一言**：访问 https://cloud.baidu.com/ 开通文心一言服务
- **阿里通义千问**：访问 https://dashscope.aliyun.com/ 开通通义千问服务
- **Google Gemini**：访问 https://aistudio.google.com/ 获取API Key
- **OneAPI**：Base URL通常为 `http://localhost:3000/v1` 或你的自定义地址

### 2. 用户Cookie配置

点击左侧菜单 **"用户Cookie配置"**：

#### 获取雪球Cookie

1. 打开浏览器，访问 https://xueqiu.com 并登录您的账号
2. 按 `F12` 键打开浏览器开发者工具
3. 切换到 **"Network"**（网络）标签
4. 刷新页面或点击任意链接
5. 在请求列表中找到任意请求
6. 点击该请求，在 **"Request Headers"**（请求头）中找到 **"Cookie"** 字段
7. 复制完整的Cookie字符串

#### 配置Cookie

1. 将复制的Cookie粘贴到输入框中
2. 点击 **"保存配置"** 按钮
3. 系统会自动验证Cookie的有效性

### 3. 评论设置

点击左侧菜单 **"评论设置"**：

| 设置项 | 说明 | 取值范围 |
|-------|------|---------|
| 每日评论限制 | 每天最多发布的评论数量 | 1-100条 |
| 最小间隔时间 | 两次评论之间的最短间隔 | 10-600秒 |
| 最大间隔时间 | 两次评论之间的最长间隔 | 10-600秒 |
| 测试模式 | 开启后不会实际发布评论 | 开关 |

**建议配置**：
- 每日评论限制：30-60条
- 最小间隔时间：30-60秒
- 最大间隔时间：60-120秒
- 首次使用建议开启测试模式验证

## 核心功能使用

### 1. 热门帖评论

1. 确保已完成基础配置（AI模型 + Cookie）
2. 点击主界面的 **"启动热门帖回复"** 按钮（绿色）
3. 系统开始自动执行：
   - 抓取雪球热门帖子
   - 使用AI生成评论
   - 发布评论到雪球
   - 按照设置的间隔时间继续处理下一篇
4. 查看实时日志了解执行情况
5. 如需停止，点击 **"停止"** 按钮

### 2. 推荐帖评论

1. 确保已完成基础配置
2. 点击主界面的 **"启动推荐帖回复"** 按钮（蓝色）
3. 后续流程与热门帖评论相同

### 3. 关注者评论/点赞

1. 点击主界面的 **"启动关注者评论"** 按钮（紫色）
2. 弹出关注者列表窗口
3. 选择要处理的关注用户（可多选）
   - 点击 **"全选"** 选择所有用户
   - 点击 **"取消全选"** 清除选择
4. 设置参数：
   - **每个用户处理帖子数**：1-20篇
   - **操作类型**：
     - 评论：生成AI评论并发布
     - 点赞：仅对帖子点赞
5. 点击 **"启动评论"** 按钮
6. 系统自动处理每个选中用户的帖子
7. 查看实时日志和统计数据

## 投资笔记功能

### 功能概述

投资笔记功能可以基于您的历史投资笔记，智能生成今日投资笔记，并支持结合最近的专栏文章作为素材。

### 使用步骤

1. **打开投资笔记窗口**
   - 在主界面找到并点击 **"撰写投资笔记"** 按钮

2. **查看历史投资笔记**
   - 系统自动弹出历史投资笔记列表窗口
   - 笔记按创建时间倒序排列（最新在前）
   - 每篇笔记显示：文章ID、标题、内容预览（前150字）

3. **查看笔记详情**
   - 点击任意笔记右侧的详情图标按钮
   - 弹出详情窗口，完整展示：
     - 文章ID
     - 完整标题
     - 全部内容
     - 实时数据：阅读量、收藏量、点赞数

4. **生成今日投资笔记**
   - 点击历史笔记列表底部的 **"生成今天的投资笔记"** 按钮
   - 系统自动筛选近5天的历史投资笔记
   - 弹出"发表雪球笔记"窗口

5. **配置生成选项**
   - **是否结合最近4篇专栏文章**（默认选择"是"）：
     - 选择"是"：将最近5天的投资笔记 + 最近4篇专栏文章作为素材
     - 选择"否"：仅使用最近5天的投资笔记作为素材
   - 点击 **"生成今日投资笔记"** 按钮

6. **编辑与确认**
   - 系统使用AI生成今日投资笔记
   - 在弹出的窗口中查看生成的内容
   - 可直接编辑修改内容
   - 确认无误后点击 **"确认发表"** 按钮

7. **发表设置**
   - 在发表窗口中预览最终内容
   - 可选择是否发表至个人专栏（默认不勾选）
   - 点击 **"发表"** 按钮完成发布

## 运行日志与监控

### 实时日志

主界面底部的 **"运行日志"** 区域实时显示系统运行状态：

- **日志风格切换**：点击日志区域上方的风格切换按钮
  - 深色风格（默认）：深色背景，清晰易读
  - 明亮风格：浅色背景，适合不同偏好
  - 两种风格字体大小和行间距完全一致

- **自动滚动**：深色风格日志会自动滚动到底部，确保看到最新内容

- **日志级别**：
  - INFO（蓝色）：一般信息
  - SUCCESS（绿色）：成功操作
  - WARNING（黄色）：警告信息
  - ERROR（红色）：错误信息

- **日志操作**：
  - 点击 **"清空日志"** 按钮清空所有日志
  - 点击 **"重新渲染"** 按钮重新显示所有日志

### 统计数据

主界面顶部显示实时统计数据：

- **已处理文章/用户数**：已处理的文章或用户数量
- **成功评论数**：成功发布的评论数量
- **失败尝试数**：失败的尝试次数
- **运行状态**：显示当前任务状态（运行中/已停止）

## 常见问题

### Q1: Cookie失效怎么办？

**A**: 重新登录雪球网站，按照"用户Cookie配置"章节的步骤获取新的Cookie并更新配置。

### Q2: AI生成评论质量不高怎么办？

**A**: 
- 尝试切换不同的AI模型
- 检查模型配置是否正确
- 不同模型的生成风格和质量可能有所不同

### Q3: 任务启动后立即停止怎么办？

**A**: 
- 检查API Key是否正确
- 检查Cookie是否有效
- 查看运行日志中的错误信息
- 先使用测试模式验证

### Q4: Gemini模型提示"high demand"错误怎么办？

**A**: 系统已内置自动重试机制，遇到高负载错误会自动重试3次，每次间隔5秒。如果仍失败，请稍后再试。

### Q5: 如何避免被雪球封号？

**A**: 
- 不要设置过高的评论频率
- 评论间隔时间建议设置在30-120秒之间
- 评论内容要有质量，避免重复
- 不要在短时间内大量评论
- 适当使用点赞功能，不要只评论
- 建议每日评论数量不超过60条

### Q6: 测试模式有什么用？

**A**: 测试模式下，系统会正常抓取帖子、生成评论，但不会实际发布到雪球。建议首次使用时先开启测试模式验证功能。

### Q7: 投资笔记生成时如何结合专栏文章？

**A**: 在"发表雪球笔记"窗口中，勾选"是否结合最近4篇专栏文章"选项（默认已勾选），系统会自动获取最近4篇专栏文章作为素材一起发送给AI模型。
